# Security Policy

Thank you for helping keep this project and its users safe.

## Supported Versions

| Version | Supported |
|---------|-----------|
| main    | Yes       |
| latest release | Yes |
| older releases | No  |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.

Use GitHub Private Vulnerability Reporting for this repository. If private reporting is not available, contact the maintainers directly at [marty@acceleratedinnovation.com].

Please include:

- A clear description of the issue
- Affected version, tag, or commit SHA
- Steps to reproduce
- Impact assessment
- Proof of concept, if safe and relevant
- Any suggested remediation

## Response Expectations

We aim to:

- Acknowledge receipt within 3 business days
- Provide an initial triage update within 7 business days
- Share progress updates for confirmed issues until resolution

## Disclosure Policy

Please allow us reasonable time to investigate, validate, and remediate the issue before public disclosure.

## Scope

In scope:

- Source code in this repository
- GitHub Actions workflows maintained in this repository
- Official release artifacts published by the maintainers

Out of scope:

- Unsupported versions
- Third-party services or dependencies not maintained in this repository
- Reports requiring social engineering, physical access, or denial of service testing

## Safe Harbor

If you act in good faith, follow this policy, avoid privacy violations, data destruction, and service disruption, we will treat your research as authorized.
