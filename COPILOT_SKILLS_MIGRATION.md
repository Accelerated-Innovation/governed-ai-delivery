# Copilot Agent: Open Agent Skills Migration

## What changed

The Copilot agent in this repo has been refactored to use the **open agent skills standard** (SKILL.md format), which GitHub Copilot added GA support for in December 2025.

Previously, governance tasks were delivered as `.prompt.md` files deployed to `.github/prompts/` in target projects. These used a Copilot-specific format (`agent: "ask"` frontmatter) and template variables like `{{FEATURE_NAME}}`.

They are now delivered as `SKILL.md` files deployed to `.github/skills/` — the same open standard already used by Claude Code (`.claude/skills/`) and Codex (`.agents/skills/`) in this repo.

## Why

- Consistency: all three agents (Copilot, Claude Code, Codex) now use the same SKILL.md format and variable syntax (`$ARGUMENTS`)
- Copilot users gain the same slash-command invocation UX (`/spec-planning <feature>`) that Claude Code and Codex users already have
- The open standard is supported by 26+ AI platforms — governance skills are no longer Copilot-proprietary

## What was removed

- `agents/copilot/prompts/` (14 `.prompt.md` files)
- `.github/prompts/` deploy target (no longer generated by `govkit apply copilot`)

## What was added

- `agents/copilot/skills/` with 14 `SKILL.md` files mirroring the Claude Code and Codex layout
- `.github/skills/` deploy target (generated by `govkit apply copilot`)

## Format changes

| Old | New |
|---|---|
| `agent: "ask"` frontmatter key | removed |
| `{{FEATURE_NAME}}` template variable | `$ARGUMENTS` |
| `<FEATURE>` / `<feature_name>` path placeholders | `$ARGUMENTS` |
| Deployed to `.github/prompts/` | Deployed to `.github/skills/` |
